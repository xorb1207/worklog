# WorkLog — 업무 자동화 앱 개발 지침

## 역할 정의

나는 이 프로젝트의 **업무 자동화 아키텍트**다.
목표는 단 하나 — **사람이 업무를 더 쉽게, 덜 까먹고, 덜 지치게** 만드는 것.

기능을 많이 만드는 게 아니라, 실제로 매일 쓰는 기능을 잘 만드는 데 집중한다.
자동화는 사용자의 판단을 대신하는 게 아니라, **빠뜨릴 것 같은 걸 옆에서 살짝 알려주는 역할**이다.

---

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목적 | 일일 업무 기록 + 아침 브리핑 + 잊혀진 Task 방지 |
| 환경 | 회사: Windows 폐쇄망 / 집: macOS |
| 알림 | Windows Toast (PowerShell) / macOS osascript |
| 데이터 | SQLite 단일 파일 (`data/worklog.db`) |
| 향후 연동 | Telegram (집) / Knox 메신저 (회사 내부망, 3단계) |

---

## 기술 스택

- **Backend**: Python 3 + Flask
- **DB**: SQLite (portable, 설치 불필요)
- **Frontend**: Jinja2 템플릿 + 순수 HTML/CSS/JS (프레임워크 없음)
- **자동화**: Python 스크립트 + OS 스케줄러 (Windows 작업 스케줄러 / macOS launchd)
- **패키징**: PyInstaller로 .exe 배포 예정 (폐쇄망용)

## 핵심 설계 원칙

### 1. 자동화는 제안, 결정은 사용자

자동화 시스템이 상태를 강제로 바꾸는 경우는 **딱 하나**만 허용한다.

| 상황 | 처리 방식 |
|------|----------|
| 마감 D-0인 TODO | **자동으로 DOING 전환** (당일 마감이라 예외 허용) |
| 마감 D-1인 TODO | 브리핑에 제안만. 사용자가 결정. |
| 7일 이상 방치 DOING | 알림만. 강제 강등 없음. |
| DOING 5개 초과 | 경고만. 강제로 막지 않음. |

### 2. 알림 & 메시지 톤 — 친절한 동료처럼

시스템 알림은 **냉정한 경고가 아니라 옆 동료가 살짝 말해주는 느낌**으로 작성한다.

```
❌ 나쁜 예 (딱딱하고 위협적)
"경고: 미완료 태스크 3건 감지됨. 즉시 처리 필요."

✅ 좋은 예 (친절하고 가볍게)
"오늘 마감인 게 1개 있어요 👀 혹시 놓치고 있진 않으셨나요?"
"일지가 아직 비어 있네요! 퇴근 전 5분만 기록해두면 내일이 편해요 😊"
"이 Task, 벌써 8일째 진행 중이에요. 아직 하고 계신 건가요?"
```

**메시지 작성 가이드**
- 단정하지 말고 **물어보는 형식**으로
- 숫자보다 **맥락**을 중심으로 (몇 개가 아니라 어떤 상황인지)
- 너무 자주 알리지 않기 — 알림 피로 방지
- 이모지는 1개만, 과하지 않게

### 3. 데이터 단순성 유지

- SQLite 파일 하나 = 전체 데이터
- 백업 = 파일 복사
- 회사 ↔ 집 동기화 = 파일 하나 옮기면 끝
- 외부 서비스 의존 없음 (폐쇄망 완전 독립)

### 4. UI 고도화는 나중에

지금은 **자동화 엔진**이 우선이다.
UI는 돌아가는 수준으로 유지하고, 엔진이 안정화된 뒤에 다듬는다.
현재 다크/라이트 테마 지원까지는 완료.

## Task 상태 정의

| 상태 | 의미 |
|------|------|
| `TODO` | 전체 백로그. 언젠가 해야 할 것. |
| `DOING` | 오늘의 포커스. 지금 하고 있는 것. (권장 최대 5개) |
| `DONE` | 완료 |
| `CANCELLED` | 취소 |

## 브리핑 구조 (scheduler.py)

### 일일 브리핑 (월~일 09:00, 단 월요일은 주간 브리핑으로 대체)

```
📅 2026-04-25 (금) 모닝 브리핑
━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔥 오늘 마감이에요! (자동으로 DOING으로 옮겼어요)
  · [A] 보고서 제출

📌 오늘 포커스 (DOING 3개)
  · [A] 보고서 제출       마감: 오늘
  · [B] 코드 리뷰         마감: 내일
  · [B] 팀 미팅 준비

✅ 어제 완료: 2개  |  이번 주 완료율: 67%

💬 이 Task, 아직 진행 중인가요? (8일째)
  · [B] 문서 정리
```

### 주간 브리핑 (월요일 09:00)

```
📊 이번 주 브리핑 (4/21 ~ 4/27)
━━━━━━━━━━━━━━━━━━━━━━━━━━━

지난 주: 완료 8개 / 미완료 3개 (완료율 73%)

이번 주 마감
  · [A] 기획서 제출  →  4/26 (목)
  · [B] 월간 보고    →  4/30 (화)

오래된 Task들, 한번 점검해보는 건 어떨까요?
  · [B] 코드 리팩토링  (14일째)
```

### 퇴근 전 체크 (17:30)

```
퇴근 전 잠깐만요 ✋

오늘 일지가 아직 비어 있어요.
5분만 투자하면 내일 아침이 훨씬 편해져요!
→ http://localhost:8080
```
