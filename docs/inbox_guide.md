# 📥 Inbox 사용법

**Inbox는 "정보가 덜 채워졌거나 아직 확정 안 된 항목"을 자동으로 모아주는 정리(triage) 화면이다.** 별도 목록이 아니라 기존 Task에서 손볼 게 남은 것만 골라 보여준다.

---

## 4가지 버킷

Inbox는 아래 4가지 기준으로 항목을 자동 분류한다. 한 항목이 여러 버킷에 겹칠 수 있고, 카드에는 해당하는 이유 태그가 색 칩으로 붙는다.

| 버킷 | 조건 | 의미 |
|------|------|------|
| **확인 필요** | `confirmed = 0` | 빠르게 던져넣어 아직 확정 안 한 항목 (봇/Quick Capture 등) |
| **미분류** | 프로젝트 없음 / `미분류` | 어느 프로젝트인지 분류 안 됨 |
| **날짜 없음** | 우선순위 A·B인데 마감일 없음 | 중요한데 마감일이 비어 있음 |
| **대기 대상 없음** | type=WAITING인데 대기 대상 비어 있음 | 누구를 기다리는지 안 적힘 |

> DONE / CANCELLED 항목은 (확인 필요 버킷을 제외하고) 집계에서 빠진다.

---

## 사용 흐름

상단에 버킷별 카운트 요약 카드가 있고, 그 아래 필터 칩(전체 / 확인 필요 / 미분류 / 날짜 없음 / 대기 대상 없음)으로 좁혀 볼 수 있다.

각 항목 카드에서 할 수 있는 액션은 3가지다.

- **✓ 확정** — 항목을 확정 처리한다 (`confirmed = 1`). "확인 필요" 버킷에서 빠진다.
- **✏️ 편집** — 카드를 더블클릭하거나 편집 버튼을 눌러 인라인 편집. 제목·타입·우선순위·프로젝트·마감일·대기 대상을 그 자리에서 채운다. `⏎` 저장 / `Esc` 취소. 빠졌던 정보를 채우면 해당 버킷 조건에서 자연히 사라진다.
- **✕ 무시** — 더 볼 필요 없는 항목을 취소 처리한다 (`CANCELLED`).

처리하면 목록이 갱신되며 그 항목은 Inbox에서 사라진다.

---

## Inbox Zero 워크플로우

목표는 **Inbox를 비우는 것**이다. 매일(또는 며칠에 한 번) Inbox를 열어 위에서부터 확정·보완·무시 중 하나로 처리해 내려가면, 모든 Task가 "확정됐고 + 분류됐고 + 마감일이 있고 + 대기 대상이 명확한" 상태가 된다. 비면 `Inbox가 비어 있어요 ✨`가 뜬다.

빠뜨릴 것 같은 걸 옆에서 살짝 알려주는 역할 — 강제하지 않고, 손볼 게 남은 것만 모아 보여줄 뿐이다.

---

## 미구현 (예정)

- **팔로업으로 전환** — 항목을 FOLLOWUP 타입으로 바꿔 후속 추적
- **관련 Task 만들기** — DECISION/ISSUE 등 기록성 항목에서 실제 할 일 Task 파생
- **Board에서 보기** — 카드에서 바로 Board 뷰로 이동
- **원문(source_text) 표시** — 자동 감지된 입력 원문 노출 (자동감지 기능 추가 시)
