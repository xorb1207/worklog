# WorkLog 사용 설명서

> 일일 업무 기록 + 아침 브리핑 + 잊혀진 Task 방지를 위한 개인용 업무 자동화 앱.

## 버전 히스토리

| 버전 | 날짜 | 변경 내용 |
|------|------|----------|
| v1.0 | 2026-06-01 | 최초 작성. 5탭 구조(Today/Inbox/Board/Review/Search), Task 입력·속성, Inbox, 단축키 문서화 |

> 기능이 추가될 때마다 이 표에 한 줄씩 추가한다.

## 1. 프로젝트 개요

WorkLog는 **사람이 업무를 더 쉽게, 덜 까먹고, 덜 지치게** 만드는 것을 목표로 한 개인용 업무 기록·관리 앱이다.

설계 철학은 단순하다 — **자동화는 제안, 결정은 사용자.** 시스템은 빠뜨릴 것 같은 일을 옆에서 살짝 알려줄 뿐, 사용자의 판단을 강제로 대신하지 않는다. (유일한 예외: 마감 당일 D-0 Task는 자동으로 DOING 전환.)

- **데이터**: SQLite 파일 하나(`data/worklog.db`)가 전체 데이터. 백업 = 파일 복사, 회사↔집 이동 = 파일 하나 옮기기.
- **환경**: 회사 Windows 폐쇄망 / 집 macOS. 외부 서비스 의존 없음.
- **구성**: Python + Flask 백엔드 + 단일 React 화면(`templates/react_app.html`).

## 2. 화면 구성 — 5개 탭

왼쪽 사이드바에서 탭을 전환한다. 사이드바 아래쪽에는 **프로젝트 필터** 목록과 **다크/라이트 모드 토글**이 있다.

### 🏠 Today
오늘 하루를 위한 메인 화면. 3개 컬럼으로 구성되며, 컬럼 순서는 ◀▶ 버튼으로 바꿀 수 있다.

- **📌 오늘 집중** — 오늘 할 일(DOING 상태). 상단에 빠른 추가 입력창이 있다. 권장 최대 5개로, 4개에서 노랑·5개에서 빨강 배지로 경고한다(강제로 막지는 않음).
- **📝 오늘 기록** — 마크다운 일지. `##` 헤더로 섹션을 접고 펼 수 있고, 슬래시 명령으로 Task를 만들 수 있다. 입력하면 약 0.6초 후 자동 저장된다.
- **📊 이번 주** — 이번 주 완료율, 최근 8주 완료 추이, 평일 완료 히트맵 등 통계.

상단의 미니 달력 / 날짜 이동으로 과거·미래 날짜의 일지도 볼 수 있다.

### 📥 Inbox
정보가 덜 채워졌거나 아직 확정 안 된 항목을 모아 처리하는 정리(triage) 화면. → 자세한 내용은 [5. Inbox 사용법](#5-inbox-사용법).

### 📋 Board
모든 Work Item을 한눈에 보는 전체 현황판. 3단계로 필터링한다.

- **타입 탭**: 전체 / 📌 Task / ⏳ 대기 / ⚠ 이슈 / ✅ 결정 / 🔁 팔로업
- **필터 바**: 프로젝트 · 상태(열린 항목 / 완료됨 / 취소됨 / 전체) · 우선순위(A / B / C)
- 목록은 **우선순위 → 마감일 → 최근 수정순**으로 정렬된다.
- 각 카드에서 상태 변경(진행/완료/대기/취소), 편집, 삭제가 가능하다.

### 🔍 Review
돌아보기·점검용 화면. 3개 하위 탭.

- **📢 Daily** — 오늘의 모닝 브리핑(오늘 마감, 포커스 항목, 방치 Task 등).
- **🧹 Cleanup** — 오래되거나 정리가 필요한 Task를 점검.
- **📅 Weekly** — 주간 완료/미완료 통계.

### 🔎 Search
`Cmd/Ctrl + K` 또는 사이드바에서 열리는 검색 모달. Task와 일지를 함께 검색하고, 결과에서 해당 날짜로 바로 이동할 수 있다.

> **참고**: 📅 전체 일정(Gantt) 뷰는 코드에 남아 있으나 현재 메뉴에서는 숨겨져 있다.

## 3. Task 입력 방법

### 방법 1 — 빠른 추가 (Today > 오늘 집중)
입력창에 제목을 쓰고 **Enter**. 옆에서 우선순위(A/B/C), 프로젝트, 마감일을 함께 지정할 수 있다.

- 우선순위 기본값은 **B**, 프로젝트를 안 고르면 현재 선택된 프로젝트 필터(없으면 `미분류`)로 들어간다.
- 오늘 집중(DOING)이 **이미 5개 이상이면** 새 Task는 DOING이 아니라 **TODO(백로그)** 로 들어간다. — 집중 한계 보호.

### 방법 2 — 일지 슬래시 명령 (Today > 오늘 기록)
일지 본문에서 `/`로 시작하면 Task를 만들면서 동시에 일지에 체크박스 줄을 남긴다.

```
/task 보고서 초안 작성 !A @기획 ~6/15
```

| 토큰 | 의미 | 예시 |
|------|------|------|
| `!A` `!B` `!C` | 우선순위 | `!A` |
| `@이름` | 프로젝트 | `@기획` |
| `~M/D` | 마감일(올해 기준) | `~6/15` |

위 명령은 일지에 `- [ ] 보고서 초안 작성 (~6/15) !A 📌 @기획` 줄을 만들고 Task로도 등록된다.

**슬래시 명령 종류** (타입별):

| 명령 (별칭) | 타입 |
|-------------|------|
| `/task` `/t` `/할일` | Task |
| `/waiting` `/w` | 대기 |
| `/decision` `/d` | 결정 |
| `/issue` `/i` | 이슈 |
| `/followup` `/f` | 팔로업 |

## 4. Task 속성

### 타입 (type)
| 타입 | 아이콘 | 마감일 라벨 | 비고 |
|------|--------|-------------|------|
| Task | 📌 | 마감 | 기본값. 추가 시 바로 진행(DOING) 시작 |
| 대기 (WAITING) | ⏳ | 확인일 | 누구를 기다리는지(`@대상`) 지정 |
| 결정 (DECISION) | ✅ | 결정일 | 기록성 항목 |
| 이슈 (ISSUE) | ⚠ | 해결목표 | |
| 팔로업 (FOLLOWUP) | 🔁 | 팔로업 | `@대상` 지정 |

### 상태 (status)
| 상태 | 의미 |
|------|------|
| `TODO` | 전체 백로그. 언젠가 해야 할 것 |
| `DOING` | 오늘의 포커스. 지금 하고 있는 것 (권장 최대 5개) |
| `DONE` | 완료 |
| `CANCELLED` | 취소 |

### 그 외 속성
- **우선순위**: A / B / C (기본 B)
- **프로젝트**: 분류 라벨. 미지정 시 `미분류`
- **마감일**: 날짜 (타입에 따라 라벨이 달라짐)
- **대기 대상**: WAITING·FOLLOWUP에서 기다리는 사람/항목
- **노트**: 자유 메모
- **확정 여부(confirmed)**: 빠르게 던져넣은 항목은 미확정 상태로 Inbox에 모임

## 5. Inbox 사용법

**Inbox는 "정보가 덜 채워졌거나 아직 확정 안 된 항목"을 자동으로 모아주는 정리(triage) 화면이다.** 별도 목록이 아니라, 기존 Task에서 손볼 게 남은 것만 골라 보여준다.

### 4가지 버킷
한 항목이 여러 버킷에 겹칠 수 있고, 카드에는 해당하는 이유 태그가 색 칩으로 붙는다.

| 버킷 | 조건 | 의미 |
|------|------|------|
| **확인 필요** | 미확정(`confirmed = 0`) | 빠르게 던져넣어 아직 확정 안 한 항목 |
| **미분류** | 프로젝트 없음 / `미분류` | 어느 프로젝트인지 분류 안 됨 |
| **날짜 없음** | 우선순위 A·B인데 마감일 없음 | 중요한데 마감일이 비어 있음 |
| **대기 대상 없음** | 대기(WAITING)인데 대상이 비어 있음 | 누구를 기다리는지 안 적힘 |

> DONE / CANCELLED 항목은(확인 필요 버킷 제외) 집계에서 빠진다.

### 사용 흐름 — 3가지 액션
상단 요약 카드와 필터 칩(전체 / 확인 필요 / 미분류 / 날짜 없음 / 대기 대상 없음)으로 좁혀 본다. 각 카드에서:

- **✓ 확정** — 항목을 확정 처리(`confirmed = 1`). "확인 필요"에서 빠진다.
- **✏️ 편집** — 카드를 더블클릭하거나 편집 버튼으로 인라인 편집. 제목·타입·우선순위·프로젝트·마감일·대기 대상을 채운다. `Enter` 저장 / `Esc` 취소. 정보를 채우면 해당 버킷에서 자연히 사라진다.
- **✕ 무시** — 더 볼 필요 없는 항목을 취소 처리(CANCELLED).

처리하면 목록이 갱신되며 그 항목은 Inbox에서 사라진다.

### Inbox Zero 워크플로우
목표는 **Inbox를 비우는 것**. 매일(또는 며칠에 한 번) Inbox를 열어 위에서부터 확정·보완·무시로 처리해 내려가면, 모든 Task가 "확정됐고 + 분류됐고 + 마감일이 있고 + 대기 대상이 명확한" 상태가 된다. 비면 `Inbox가 비어 있어요 ✨`가 뜬다.

## 6. 단축키 / 인터랙션

`?` 키를 누르면 앱 안에서 단축키 도움말을 볼 수 있다. (Mac은 `⌘`, Windows는 `Ctrl`)

| 단축키 | 동작 |
|--------|------|
| `Cmd/Ctrl + K` | 검색 열기/닫기 |
| `Cmd/Ctrl + N` | 빠른 추가 입력창에 포커스 |
| `Cmd/Ctrl + J` | 일지에 포커스 |
| `Cmd/Ctrl + ←` | 하루 전 일지로 |
| `Cmd/Ctrl + →` | 하루 후 일지로 |
| `Cmd/Ctrl + D` | 다크모드 토글 |
| `?` | 단축키 도움말 열기/닫기 |
| `Esc` | 모달 닫기 |

**일지 편집 중**
- `Cmd/Ctrl + B` — 선택 텍스트 굵게
- `Cmd/Ctrl + I` — 선택 텍스트 기울임
- `Enter` — 빠른 추가/슬래시 명령 확정 (한글 조합 중에는 무시)

**카드 인라인 편집 (Inbox · Board)**
- 카드 **더블클릭** 또는 ✏️ 버튼 → 편집 모드
- `Enter` 저장 / `Esc` 취소

**일지**
- `##` 헤더 섹션 접기/펼치기, 전체 접기/펼치기
- 체크박스 `- [ ]` 클릭으로 완료 토글
- 입력 후 약 0.6초 뒤 자동 저장

## 7. 알려진 제한사항 / 미구현 예정

**Inbox 2단계 (카드에 자리만 있고 미구현)**
- 팔로업으로 전환 — 항목을 FOLLOWUP 타입으로 바꿔 후속 추적
- 관련 Task 만들기 — 결정/이슈 등 기록성 항목에서 실제 할 일 파생
- Board에서 보기 — 카드에서 바로 Board 뷰로 이동
- 원문(source_text) 표시 — 자동 감지 입력 원문 노출 (자동감지 기능 추가 시)

**기타**
- 📅 전체 일정(Gantt) 뷰 — 코드는 있으나 메뉴에서 숨김 상태
- 월간 보고서 자동 초안(.docx) 생성 — 로드맵 예정
- PyInstaller `.exe` 패키징(폐쇄망 배포) — 로드맵 예정
- Knox 메신저 연동(회사 내부망) — 3단계 예정
